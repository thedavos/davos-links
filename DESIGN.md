# Davos Links Design

Davos Links should look and feel like a precise developer tool.

## Direction

- Vercel-inspired product interface.
- Geist Sans for primary UI text.
- Geist Mono for IDs, short paths, timestamps, URLs, and metrics.
- Black, white, and neutral gray foundation.
- Crisp borders, compact spacing, and exact alignment.

## Principles

- Prioritize speed and clarity over decoration.
- Put the link workflow first.
- Make table scanning easy.
- Keep actions predictable and close to the data they affect.
- Use motion sparingly and only for state changes.

## Layout

The dashboard should use:

- A restrained top navigation or sidebar.
- A primary links table.
- A clear create-link action.
- Compact stat summaries.
- Detail/edit views that keep destination, short path, status, and analytics visible.

Avoid nested cards, oversized marketing sections inside the app, and decorative backgrounds.

## Components

- Buttons should use clear labels or recognizable icons.
- Inputs should show validation messages near the field.
- Tables should support empty and loading states.
- Copy actions should use icon buttons with clear tooltips or labels.
- Destructive actions should require confirmation before production hardening.
- Disabled states should explain what is missing when practical.

## Visual System

- Base background: white.
- Text: near black and neutral grays.
- Borders: neutral gray.
- Accent usage: minimal and functional.
- Radius: small and consistent.
- Typography: compact, with clear hierarchy.

## Content

Use direct product language:

- "Create link"
- "Short path"
- "Destination URL"
- "Copy"
- "Deactivate"
- "Clicks"

Avoid explanatory marketing copy in the dashboard. The user is there to work.
